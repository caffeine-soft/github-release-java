# GitHub Release Java

A lightweight, zero-dependency GitHub Action to create GitHub Releases and upload artifact assets natively in Java 25.

## Usage

Add the following step to your GitHub Actions workflow.

```yaml
name: Release
on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write # Required to create releases and upload assets
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Build App
        run: ./gradlew build

      - name: Create GitHub Release
        id: create_release
        uses: caffeine-soft/github-release-java@v1
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          tag: ${{ github.ref_name }}
          name: Release ${{ github.ref_name }}
          body: 'Automated release generated by GitHub Actions.'
          artifacts: 'build/libs/*.jar, dist/*.zip'
          draft: false
          prerelease: false
```

## Inputs

| Input | Description | Default | Required |
|-------|-------------|---------|:--------:|
| `github_token` | Token used to authenticate with the GitHub API. | `${{ github.token }}` | **Yes** |
| `tag` | The tag name for the release. | | **Yes** |
| `name` | The name of the release. | Falls back to `tag` | No |
| `body` | Text describing the contents of the release. | Auto-generated | No |
| `draft` | Creates a draft release if set to `true`. | `false` | No |
| `prerelease`| Identifies the release as a prerelease if set to `true`.| `false` | No |
| `artifacts` | Comma-separated list of file paths or glob patterns to upload (e.g. `build/libs/*.jar`). | `<empty>` | No |

## Outputs

| Output | Description |
|--------|-------------|
| `release_id` | The internal GitHub ID of the newly created release. |
| `release_url`| The HTML URL of the created release page. |

## Compatibility
This action requires the GitHub Runner to have Java 25 installed. If you are not using `ubuntu-latest` (which may update its default Java version over time), or if your runner lacks Java 25, ensure you run `actions/setup-java@v4` with `java-version: '25'` before this step.
